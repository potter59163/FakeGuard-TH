## แหล่งที่มา

1. **ศูนย์ต่อต้านข่าวปลอม (Anti-Fake News Center) — พอร์ทัล Open Data ทางการ**
   `https://opendata.antifakenewscenter.com` (endpoint `/api/export-posts` ตัวเดียวกับ
   ปุ่มดาวน์โหลด CSV บนเว็บ) ดึงทีละเดือน 60 เดือน rate-limit 1.5 วินาที/request
   ได้ข้อมูลดิบ 11,286 รายการ
2. **AFP Fact Check Thailand** — เข้าถึงอัตโนมัติไม่ได้ (เว็บมี bot protection ตอบ 403)
   จึงยังไม่ได้รวมในรอบนี้ → บันทึกเป็นข้อจำกัด และเป็นงานต่อยอด (เก็บผ่านเบราว์เซอร์จริง)

## นิยาม Label

- `1` = ข่าวปลอม (สถานะ "ข่าวปลอม" จาก AFNC)
- `0` = ข่าวจริง (สถานะ "ข่าวจริง" จาก AFNC)
- **ตัดทิ้ง**: "ข่าวบิดเบือน" (ผสมจริง+เท็จ ทำให้ label ไม่สะอาด), "คลังความรู้",
  "อาชญากรรมออนไลน์", "ข่าวอื่นๆ" — เหตุผล: โจทย์เป็น binary classification
  ต้องการ label ชัดเจนสองขั้ว

## การป้องกัน Data Leakage (จุดสำคัญของระเบียบวิธี)

หัวข้อข่าวของ AFNC มี "คำเฉลย" ฝังอยู่ เช่น ข่าวปลอมขึ้นต้นด้วย
"ข่าวปลอม อย่าแชร์!" และข่าวจริงลงท้ายด้วย "จริงหรือ?" — ถ้าปล่อยไว้
โมเดลจะเรียนรู้คำเฉลยแทนเนื้อหาข่าว (ได้คะแนนสูงเกินจริงแบบไร้ความหมาย)

มาตรการ:
1. ตัด prefix เฉลยทุกรูปแบบ: "ข่าวปลอม/ข่าวบิดเบือน/ข่าวจริง", "อย่าแชร์!",
   "แชร์ว่อน!!", "สร้างความปั่นป่วน หยุดแชร์!" (ทำซ้ำจนไม่เหลือ)
2. ตัด suffix เฉลย: "จริงหรือ?", "จริงหรือไม่", "ใช่หรือไม่", "ไม่ควรแชร์ต่อ"
3. แถวที่ยังมีคำเฉลยตกค้างหลังตัด → ทิ้งทั้งแถว (40 rows)
4. ตรวจแล้วความยาวเฉลี่ยสองคลาสใกล้กัน (จริง 69.5 / ปลอม 64.0 ตัวอักษร)
   → ไม่มี feature ตื้นๆ ให้โมเดลโกง

## การกรองอื่นๆ

- ตัดข้อความสั้นกว่า 15 ตัวอักษร (ไม่มีเนื้อหาพอ)
- Dedupe ข้อความซ้ำ (case-insensitive exact match หลัง normalize NFC + ยุบช่องว่าง)
- Normalize Unicode NFC, ยุบช่องว่างซ้ำ

## ข้อจำกัด

1. **ข้อความเป็น "หัวข้อข่าว" สั้นๆ** (เฉลี่ย ~65 ตัวอักษร) ไม่ใช่โพสต์โซเชียลเต็ม
   → ผลอาจ generalize ไปโพสต์ยาวๆ ได้จำกัด
2. **Label มาจาก AFNC แหล่งเดียว** (AFP เข้าไม่ได้) → มี selection bias ตามหัวข้อ
   ที่ AFNC เลือกตรวจสอบ (เน้นสุขภาพ/นโยบายรัฐ/การเงิน)
3. ข่าวจริงใน AFNC คือ "ข่าวที่คนสงสัยแล้วตรวจว่าจริง" ไม่ใช่ข่าวจริงทั่วไป
   → เป็น distribution เฉพาะ (แต่ก็ตรงกับ use case จริง: ข้อความที่คนไม่แน่ใจ)
4. สัดส่วนใน pool ไม่สมดุล (ปลอม 73%) → แก้โดยสุ่ม 500:500 (seed=42)

## สัญญาอนุญาต / จริยธรรม

- ข้อมูลจากพอร์ทัล Open Data ของหน่วยงานรัฐ เผยแพร่สาธารณะเพื่อการใช้งานทั่วไป
- ไม่มีข้อมูลส่วนบุคคล (เก็บเฉพาะหัวข้อข่าว + หมวด + วันที่)
- เก็บด้วย rate limit 1.5 วิ/request พร้อมระบุตัวตนใน User-Agent
